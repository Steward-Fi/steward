## Release Gate Order

1. Run API/security rows first; they catch secret leakage and auth bypasses fastest.
2. Run SDK/React contracts after API routes settle.
3. Run browser E2E only after the web app builds and the local API fixture is stable.
4. Finish with `cd packages/api && /Users/shawwalters/.bun/bin/bun run build`, package builds, then root `bun run verify` once concurrent edits are merged.

## Known Watch Areas

- Multi-HD/indexed user-wallet work is still partial. The current slice covers `walletIndex`/`wallet_index` selectors for selected user-wallet API and SDK operations, legacy-compatible index 0 behavior, deterministic positive-index user-wallet agent IDs, indexed EVM/Solana derivation for recoverable restore, invalid/conflicting selector rejection with audit evidence on sensitive routes, and aggregate user-wallet-family spend/rate policy evaluation for signing. Do not mark full Privy parity until pregenerated multi-HD wallets, full dashboard selection/management UX, and richer per-wallet policy/signer workflows are implemented and e2e-tested.
- PGLite-backed suites are slower and can be noisy when run in a single process; prefer exact-file commands above.
- `vault-mfa-sensitive-actions.test.ts` and other PGLite route suites should not be grouped with unrelated PGLite suites in one Bun invocation; cross-file teardown can close the shared override before later route requests complete.
- Webhook dispatch mock tests may pass while logging expected async dispatch errors if the local mock intentionally omits full Drizzle chaining.
- Full repo `biome check` and root builds are not authoritative while unrelated product files are actively dirty.
